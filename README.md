# Simulation Ecosim

Проект представляет собой симуляцию, построенную на комбинации **data-driven подхода, ECS-подобной обработки и объектно-ориентированной логики поведения**.

В основе лежат следующие решения:  
`EntryPoint`, `DI (Zenject)`, `Object Pooling`, `FSM`, `Specification (Entity)`, `Factory`

---

## Общая архитектура

Архитектура каждой сцены строится вокруг локальной **FSM (Finite State Machine)**.  
Состояния определяют жизненный цикл сцены и управляют всем, что происходит в игровом мире — от инициализации до игрового цикла.

FSM собирается через **Zenject** на уровне Scene Installer, где регистрируются состояния и зависимости. После инициализации управление полностью передаётся активному состоянию.

Вход в сцену происходит через **EntryPoint** — MonoBehaviour, размещённый на объекте сцены.

Он отвечает за: инициализацию DI-контейнера, запуск FSM, старт основных систем  

---

## Сущности и их создание

Все игровые сущности представлены базовым классом `Entity`, который выступает контейнером состояния и точкой интеграции поведения.

Создание сущностей происходит через **Factory**, работающую на основе набора спецификаций.  
Каждый тип юнита конфигурируется отдельно через эти спецификации.

```csharp
[CreateAssetMenu(menuName = "Entity/Specification", fileName = "EntitySpecification")]
public class EntitySpecification : ScriptableObject
{
    [field: SerializeField] public string Name { get; private set; }
    [field: SerializeField] public EntityType Type { get; private set; }

    [Space, SerializeField] public BehaviourSpecification Behaviour;
    [SerializeField] public List<Specification> Configuration;
}
```

Регистрация и управление жизненным циклом сущностей осуществляется через **Object Pool с предварительной загрузкой**.

Это позволяет избежать лишних аллокаций во время выполнения, переиспользовать объекты и задействованные спецификации и повысить производительность при большом количестве сущностей  

---

## Specification подход

В проекте используется разделение спецификаций на два типа.

### Описательные спецификации

Определяют структуру сущности:
- наличие и параметры `NavMeshAgent`  
- коллайдеры  
- визуальную модель  
- `Transform`  
- базовые характеристики
- ...

Фактически это **data-driven описание**, определяющее, *что из себя представляет сущность*.

---

### Поведенческие спецификации

Отвечают за создание логики и реализованы через принципы **инкапсуляции и полиморфизма** на основе системы задач.

Каждая спецификация:
- инкапсулирует параметры поведения  
- создаёт конкретную реализацию  
- возвращает её в виде интерфейса  

```csharp
public override IBehaviour Create()
{
    return new WorkerBehaviour(_speed, _eatToSplit, _eatDuration, _distanceToEat);
}
```
